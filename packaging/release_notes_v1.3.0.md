## LiveBabel v1.3.0

本地优先的语音工具箱，支持实时字幕、离线字幕、会议纪要、语音输入和文本朗读。

### ✨ 本次更新

- **新增：ChatTTS 文本朗读** — 生成会议纪要后可直接朗读全文，采用语义分段和分块播放，长文本无需等待全部合成完成。
- **优化：自然连续的朗读体验** — 合并过短句段，优先按中文标点切分，并复用固定说话人参数，减少逐句音色和语气跳变。
- **新增：ChatTTS 模型按需下载** — 首次使用朗读时单独提示下载约 470 MB 模型，不再把可选朗读模型绑定到首次启动的必要模型下载中。
- **更新：ChatTTS 专用 sherpa-onnx wheel** — Windows Python 3.11 默认使用已验证的 CPU wheel；ChatTTS 默认走 CPU，以获得更稳定的实际推理速度。
- **优化：语音输入改为右 Ctrl PTT** — 按住键盘右侧 Ctrl 说话，松开结束并输入；移除容易误触的双击常开模式。
- **优化：语音输入反馈** — 按下后立即显示“正在聆听…”，松开后显示“正在整理文字…”，定稿完成后短暂展示最终文本。
- **修复：连续听写稳定性** — 每轮开始和结束都会清理识别状态，避免上一轮草稿或 VAD 状态污染下一轮。
- **修复：松开后无定稿文字** — SenseVoice 未返回尾部 final 时保留有效流式草稿；后台定稿异常也会正确恢复，不再永久卡在整理状态。
- **修复：快速松开和关闭竞态** — 改善麦克风初始化、停止、异步定稿与最终注入之间的线程时序。

### 下载

本版本优先提供 **Windows CPU 版**：

- `LiveBabel-CPU-v1.3.0-win64.zip`

无需 NVIDIA 显卡，支持 Windows 10 / 11 64 位。模型不随程序包提供，首次使用对应功能时由程序按需下载。

macOS arm64 版使用独立的 `v1.3.0-mac` Release；语音输入目前仍仅支持 Windows。

### 首次使用

1. 解压 ZIP 后运行 `LiveBabel-CPU.exe`。
2. 首次启动按提示下载必要的语音识别模型。
3. 首次使用朗读时，程序会另行询问是否下载 ChatTTS 模型。
4. 翻译功能需要在主页填写 DeepSeek API Key；不填写仍可使用识别、转录和朗读。
